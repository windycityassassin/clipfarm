What it does

Most short-form video on TikTok / YouTube Shorts / Instagram Reels is repurposed long-form. Clipping a four-hour stream by hand is brutal. clipfarm turns that into clipping monitor <channel>: streamlink pulls the HLS feed, ffmpeg writes rolling MPEG-TS segments, a scoring pass identifies the peaks, and each peak gets cropped to 9:16 with a JSON sidecar of the score breakdown.

Pipeline
01
live-detect & ingest
streamlink probes the channel. On offline, retry every 60s. On live, pipe the HLS stream into ffmpeg -f segment, write 30-second TS files into a rolling buffer.
02
chat firehose
Anonymous IRC connection as justinfanNNNN. PRIVMSG timestamps go into a bisect-indexed deque so binning queries are O(log n).
03
score windows
Extract audio to mono wav (16kHz), compute RMS-in-dBFS per bin. Z-score audio + chat against their recent baselines. Combine: α·audio_z + β·chat_z.
04
non-max suppression
Sort candidates by score, keep top peaks at least min_gap_seconds apart. Re-sort chronologically for cut order.
05
cut + reformat
For each peak, find overlapping segments, concat-demux them, cut [t-pre, t+post], center-crop 9:16, encode h264+aac.
06
review queue
MP4 + JSON sidecar drop into out/<channel>/. You review and decide what to post. Phase 1 stops here.

No API keys, no OAuth. Chat is read over Twitch's anonymous IRC bridge. Streamlink handles HLS auth automatically.

What this is not

Not a publisher. Phase 1 stops at MP4-on-disk by design. Auto-upload to YouTube Shorts / TikTok / Instagram Reels is Phase 2+ and will be opt-in per channel, behind each platform's official API.

Not a stealth tool. Streamlink uses Twitch's normal viewer flow; chat is read over the public anonymous IRC bridge. If Twitch ever rate-limits or shapes the request, that's the honest answer — the tool does nothing to hide.
